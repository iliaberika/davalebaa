Global updates

GLOBAL UPDATE updates data that is stored in global storage area and that was created or updated with the GLOBAL CREATE command, using the same KEY value. The storage area is uniquely identified by the value that is specified on the KEY option. The data can be accessed for read with the GLOBAL READ command. Options DATA(data-area) Specifies a data area containing the data to be updated. KEY(keyname) Specifies the symbolic name (1-8 characters) of the KEY value to be read. This key uniquely identifies data that is written on global storage The keyname must be alphanumeric characters. KEYLENGTH(data-value) Specifies, as a halfword binary value, the length of the key named in the KEY option . FLENGTH(data-value) Specifies, as a 32-bit binary value, the number of bytes of storage area to be updated. LENGTH has an upper limit of 268435456 bytes (256 MB). If the requested length is less than zero, the LENGERR condition occurs;
